<div align="center">
<h1>Cisco Vision SNMP2MQTT</h1>
</div>

Cisco Vision SNMP2MQTT is the SNMP polling backend for Cisco Vision. It can run from a normal SNMP2MQTT targets file, or it can import the review-only YAML generated by Cisco Vision Discovery.

## v0.6.1 import-prep

Cisco Vision Discovery v0.6.0 can generate:

```text
/share/cisco_vision/generated-snmp2mqtt.yaml
```

This add-on can import that file when explicitly enabled. Import is disabled by default.

## Safe import options

```yaml
use_cisco_vision_generated_yaml: false
cisco_vision_generated_yaml_path: /share/cisco_vision/generated-snmp2mqtt.yaml
imported_targets_path: /config/addons_config/cisco_vision_snmp2mqtt/imported/generated-snmp2mqtt.yaml
backup_existing_config: true
```

When `use_cisco_vision_generated_yaml` is `false`, the add-on uses the normal `targets_path`.

When `use_cisco_vision_generated_yaml` is `true`, the add-on validates the generated file before using it. It rejects the file if:

```text
- the file is missing
- the Cisco Vision generated YAML header is missing
- the Cisco Vision Discovery source header is missing
- CHANGE_ME is present
- the targets: block is missing
- no target host entries are present
```

If validation passes, the add-on backs up the existing targets file, copies the generated YAML to `imported_targets_path`, then starts SNMP2MQTT from that imported targets file.

## Standard configuration

```yaml
mqtt:
  host: localhost
  port: 1883

targets_path: /config/addons_config/cisco_vision_snmp2mqtt/targets.yaml

use_cisco_vision_generated_yaml: false
cisco_vision_generated_yaml_path: /share/cisco_vision/generated-snmp2mqtt.yaml
imported_targets_path: /config/addons_config/cisco_vision_snmp2mqtt/imported/generated-snmp2mqtt.yaml
backup_existing_config: true

homeassistant:
  discovery: false
  prefix: homeassistant
```

For the first Cisco Vision generated YAML test, set:

```yaml
use_cisco_vision_generated_yaml: true
homeassistant:
  discovery: true
  prefix: homeassistant
```

## Expected log messages

A successful import should show:

```text
Cisco Vision generated YAML import is enabled.
Cisco Vision generated YAML validated and imported to:
  /config/addons_config/cisco_vision_snmp2mqtt/imported/generated-snmp2mqtt.yaml
SNMP2MQTT Start
```
